## 📌 Convention

**commit convention** <br>
`conventionType: 구현한 내용` <br><br>


**convention Type** <br>
| convention type | description |
| --- | --- |
| `feat` | 새로운 기능 구현 |
| `chore` | 기초 세팅 및 gradle 파일 추가 |
| `docs` | 문서 추가 및 수정, 삭제 |
| `fix` | 버그 수정 |
| `rename` | 파일 및 폴더 이름 변경 |
| `test` | 테스트 코드 추가 및 수정, 삭제 |
| `refactor` | 코드 리팩토링 |
| `style` | 부수적인 코드 수정 |

### 
- `컨벤션명/#이슈번호`

- 부득이하게 develop branch에 직접 commit 해야 할 경우, `!hotfix:` 사용
